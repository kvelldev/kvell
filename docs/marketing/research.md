
# Kvell: 市場規模算定と初期KPI策定レポート

v0.1

## 1\. エグゼクティブ・サマリー

本レポートは、個人開発サービス「Kvell」のターゲット市場規模（TAM）およびローンチ初期の重要業績評価指標（KPI）を定義するものである。

既存SNS（X, 5ch）における「サイレントマジョリティ（B層）」の不満を解消する受け皿として、**約5.7万人の潜在顧客**が存在すると推計される。
初期フェーズにおいては、規模の拡大よりも「コミュニティ密度」と「定着率」を最優先とし、**DAU 50〜100人** の高エンゲージメント集団の形成を目指す。

## 3\. 市場規模算定 (Fermi Estimation)

X（旧Twitter）のフォロワー数をベースに、Kvellが獲得可能な最大市場規模（TAM）を算出する。

| フィルタリング項目 | 係数 | 残存人数 | 算定根拠 |
| :--- | :---: | :--- | :--- |
| **① 母集団 (Base)** | - | 1,200,000人 | 櫻坂46公式Xフォロワー数 (2025年時点概算) |
| **② アクティブ率** | 50% | 600,000人 | bot、海外勢、休眠垢、他界済み層を除外。「現在情報を追っている」実数。 |
| **③ B層含有率** | 95% | 570,000人 | A層（現場全通・常時発信のノイジーマイノリティ）を5%と仮定(パレートの法則的には2割だが、SNSで「うるさい」と感じるほどの熱量は5%ほどと考えられる)。残る95%がサイレントマジョリティ。 |
| **④ 潜在ニーズ率** | 10% | 57,000人 | 現状の環境（X/5ch）に不満を持ち、新しい「居場所」を探している層の割合。 |

### 結論：TAM (Total Addressable Market)

**約 5.7万人**

* **考察:** 個人開発のBBSとして、5.7万人は十分すぎる規模である。
* **成功ライン:** このうち **1% (570人)** が常駐すれば、コミュニティとしては「大盛況」となり、サーバー負荷やモデレーションの限界値に達するレベルである。したがって、これ以上の市場拡大を狙う必要はない。

## 4\. ローンチ初期KPI (Phase 1: Ignition)

リリース後1ヶ月間（着火期）における目標指標。「過疎に見えない最低ライン」 と 「質の高い常連の確保」 に焦点を当てる。

### ■ 重要指標

| 指標 | 目標値 | 設定意図・根拠 |
| :--- | :--- | :--- |
| **DAU**<br>(Daily Active Users) | **50〜100人** | 教室2〜3個分の人数。これ以下だと「誰もいない」と感じられ、これ以上だと初期の治安維持が困難になる適正値。 |
| **WAU**<br>(Weekly Active Users) | **300人** | 菊池さんは「毎日」ではなく「週1回（番組視聴時）」訪れる特性があるため、DAUよりWAUを重視する。 |
| **1人あたり滞在時間** | **1分以上** | 「ふらっと見て帰る」ではなく、過去の焚き火（ログ）を一通り読み、ニヤニヤして共感するまでの所要時間。 |
| **焚き火成立数** | **3本 / 週** | 「そこさく実況」「公式ブログ更新」「その他（妄想等）」で、週に3つ盛り上がる場所があれば、ユーザーは「生きているサイト」と認識する。 |

### ■ 健全性指標（Quality Control）

* **リテンション率 (翌週継続率): 40%以上**
  * 新規流入数よりも、「先週来た菊池さんが、今週も戻ってきたか」を最重要視する。
* **「追焚き」発生率: 閲覧者の 5%**
  * 95%はROM専でよい。5%のユーザーが薪をくべれば、コンテンツは自律的に生成される。

## 5\. 戦略的示唆

* **ニッチの肯定:**

  * 5.7万人という数字は、スタートアップ企業としては小さいが、「個人の隠れ家」としては巨大である。マスに向けた機能追加や広告は不要。

* **密度の優先:**

  * 初期は「広く薄く」ではなく「狭く濃く」攻める。日曜深夜〜月曜夜にトラフィックを集中させる施策（実況タイムの告知など）が有効。

* **勝利条件:**

    > **「月曜の夜、50人の菊池さんが無言で集まり、1分間ニヤニヤして帰っていく」**

この光景が作れた時点で、本プロジェクトはPMF（Product Market Fit）を達成したとみなす。
